# Phase 11 着手計画

## 背景

Phase 10 で Salesforce 構築物の主要 15 メタデータ種が同一品質で Markdown 化される状態に到達。残スコープのうち、**モダン UI レイヤ** (LWC / Aura / FlexiPage) を Phase 11 で取り込む。

LWC は近年の Salesforce UI 開発の標準。Apex / Flow / Trigger からの呼び出し関係を可視化するには UI 側からの参照も必要。

## 判断 — Phase 11 のスコープ

| 区分 | サブ | やること | 優先理由 |
|---|---|---|---|
| **モダン UI** | 11-A | LWC bundle 取り込み | あらゆる新規開発の標準。Apex 呼び出し関係の起点 |
| **Legacy UI** | 11-B | Aura bundle 取り込み (軽量) | Aura は移行対象だが既存組織には残る。構成把握まで |
| **Lightning Page** | 11-C | FlexiPage 取り込み | Lightning App Builder のレコードページ。配置されるコンポーネントを可視化 |

## スコープ外 (Phase 11.x 以降)

- **Lightning App** (`*.app-meta.xml`): Lightning App の取り込み (実用度中)
- **再現性 CI**: `/yohaku-explain` を温度 0 / プロンプトハッシュ / N-run 一致でテスト
- **横断ドキュメント**: 権限マトリクス / 自動化マトリクス / ER 図
- **Profile field permission の折り畳み**

## 設計方針

### LWC の多ファイル構造への対応

LWC bundle は **ディレクトリ単位** (1 component = 4 ファイル):

```
force-app/main/default/lwc/myComponent/
  myComponent.js
  myComponent.html
  myComponent.css           (optional)
  myComponent.js-meta.xml   (← 起点)
```

実装方針:
- `.js-meta.xml` を adapter classifier の起点にする (Apex .cls-meta.xml と類似)
- extractor は **projectRoot から兄弟ファイル** (`<name>.js`, `<name>.html`, `<name>.css`) を読む
- 兄弟ファイル不在は許容 (ベストエフォート)

### LWC で抽出する情報

JS ファイルから regex ベースで:
- `import method from '@salesforce/apex/<Class>.<method>'` → Apex 呼び出し参照
- `import label from '@salesforce/label/<Label>'` → Custom Label 参照
- `@api propName` → 公開プロパティ
- `@wire(adapter, params)` → wire 呼び出し
- `dispatchEvent(new CustomEvent('eventName'))` → 発火イベント

HTML テンプレートから:
- `<c-foo-bar>` → 子 LWC
- `<lightning-*>` → 標準コンポーネント (件数のみ)
- `lwc:if` / `lwc:for-each` → ディレクティブ件数

メタ XML から:
- `apiVersion` / `isExposed` / `masterLabel` / `description` / `targets`

### Aura の軽量取り込み

bundle 構造:
```
force-app/main/default/aura/myComp/
  myComp.cmp / .app / .evt   (markup)
  myCompController.js
  myCompHelper.js
  myComp.css
  myComp.cmp-meta.xml        (← 起点)
```

抽出は**最低限**: ファイル構成、apiVersion、masterLabel、description、type (`Component` / `Application` / `Event`)。詳細解析 (XML 内の `<aura:attribute>` 等) は Phase 11.x 以降。

### FlexiPage

`*.flexipage-meta.xml` から:
- pageTemplate / type / sobjectType
- regions: [{ name, type, items: [{ componentName, fieldName? }] }]
- platformActionList などは要件次第

