# Phase 15 着手計画

## 背景

Phase 14 完了 ADR の残課題:

> 実 AI 推論との end-to-end: claude-code 環境で `/sfai-explain` slash command を実機実行し、explain-writer subagent が DETERMINISTIC のみから自然文を生成するフローの検証

Phase 14 では擬似テキスト (人間が書いた JSON) を `sfai explain-write` に流して書き戻し基盤の動作確認を行った。Phase 15 では **本物の AI 推論** (= claude-code セッション内で動く私自身が explain-writer の指針に従って文面を生成) を使って、以下を検証する。

## 判断

Phase 15 を **実機検証フェーズ** として実施し、実装変更は最小限に抑える。検証で見つかった課題のみコード修正する。

## 検証手順

各エンティティに対して以下を実施:

1. **ファクト収集**: `sfai graph query` で対象エンティティの DETERMINISTIC ファクトを取得
2. **Markdown 確認**: `docs/generated/<subdir>/<fqn>.md` の DETERMINISTIC ブロックのみを読む (AI_MANAGED / HUMAN_MANAGED は読まない)
3. **AI 推論**: explain-writer subagent の指針に従って、各 block-id の本文を生成
4. **JSON 出力**: `/tmp/sfai-explain-test/<entity>.json` に書き出し
5. **書き戻し**: `sfai explain-write --kind <kind> --fqn <fqn> --input <json>` を実行
6. **結果確認**: 書き戻された Markdown を読んで「設計書として読めるか」評価
7. **再 sync 検証**: `sfai sync` を再実行し、AI_MANAGED 内容が保全されることを確認

## 検証対象 (5 エンティティ)

| # | kind | fqn | 選定理由 |
|---|---|---|---|
| 1 | apexClass | `ClaimService` | 受注関連の業務クラス、Phase 14 では未着手 |
| 2 | apexClass | `AccountBalanceService` | Phase 14 で擬似テキストで書いた → 再生成性の比較対象 |
| 3 | apexTrigger | (graph から検出する) | Trigger 種別の検証 (Phase 14 では未着手) |
| 4 | object | `Invoice__c` | Object 種別、ER 関係性が豊富 |
| 5 | flow | `Order_AutoCreateShipmentOnApproval` | Phase 14 で擬似テキストで書いた → 再生成性の比較対象 |

## 評価観点

| 観点 | 評価方法 |
|---|---|
| **ファクト整合性** | DETERMINISTIC の数値・名前と AI 文面が矛盾しないか |
| **捏造防止** | 知識グラフに無い情報を書いていないか (推測には「(推測)」付与) |
| **設計書らしさ** | 「3 ヶ月後の自分が読んで業務理解できるか」(主観評価) |
| **形式** | Markdown 表崩れ / マーカー破損なし |
| **再生成性** | Phase 14 の擬似テキストとの比較 (同じ事実から似た文面が出るか) |
| **HUMAN_MANAGED 不可侵** | 人間の聖域に書き込んでいないか |

## ゴール条件
