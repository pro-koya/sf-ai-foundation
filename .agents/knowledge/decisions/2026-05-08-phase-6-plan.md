## 主要設計判断

### Claude Code Plugin 形式

`claude-plugin/plugin.json` を最小整備し、**Git clone 経由のインストール手順** を併記する保守的な配布戦略 (Plugin 仕様の流動性に対応):

```json
{
  "name": "@yohakuforce/yohaku",
  "version": "0.1.0",
  "description": "...",
  "compat": {
    "claude-code": ">=1.0"
  },
  "scaffold_source": "../scaffold",
  "agents_source": "scaffold/.claude/agents",
  "commands_source": "scaffold/.claude/commands",
  "hooks_source": "scaffold/.claude/settings.json"
}
```

実際の Plugin インストールフローは Claude Code の現行仕様に依存するため、**Git clone + `yohaku init --bootstrap`** を主導入経路とし、Plugin 形式は補助。

### DX MCP アダプタ層 (6-2)

[DX MCP アダプタ層 ADR](./2026-05-07-dx-mcp-adapter-pattern.md) で Phase 6 で実装と決めた件。
ただし MCP 仕様は Beta で流動的なため、**SourceAdapter インタフェース互換の stub** を実装し、実 MCP 接続は CLI の `--source dx-mcp` フラグで明示的に opt-in する設計:

```typescript
class DxMcpSourceAdapter implements SourceAdapter {
  readonly kind = "dx-mcp" as const;
  async list(): Promise<readonly MetadataDescriptor[]> {
    throw new Error("DxMcpSourceAdapter is not yet wired to a live MCP server. Phase 7 で本実装予定。");
  }
  async loadContent(): Promise<string> { throw ... }
  async computeSourceHash(): Promise<string> { throw ... }
}
```

→ コアの `LocalSourceAdapter` は不変、`--source dx-mcp` を指定したときのみ DxMcpSourceAdapter が選択される。Beta 仕様変動に **コアが影響を受けない** 設計。

### 3 profile 定着

Phase 2-6 で profile_defaults を 3 回更新したが、**Phase 6 で最終確定** する。

| profile | enabledCommands | enabledAgents | 想定対象 |
|---|---|---|---|
| minimal | onboard / explain / impact | graph-querier / object-documenter | 中小 / 試用 |
| standard | + classify-diff / change-summary | + onboarding-guide / review-assistant / classifier 5 種 | 通常運用 |
| full | + release-prep / manual-steps | + release-advisor / customer-impact-explainer / manual-step-extractor / release-composer / rollback-drafter | 大企業 / ベンダー本格運用 |

### npm 公開準備

実 publish はしない (Phase 7 で OSS 公開時)。形を整える:
- `CHANGELOG.md` (新規) — Phase 1〜6 の歴史
- `SECURITY.md` 拡充 (現状最小限) — 脆弱性報告フローの実運用化
- `package.json` メタ充実 — keywords / homepage / files

### Antigravity 互換

完全な Antigravity プラグイン化は Phase 7。Phase 6 では **互換性方針の明文化** のみ:
- `scaffold/AGENTS.md.eta` 冒頭に「Claude Code 専用ではない」注記
- prompts/ と subagent system prompt が Antigravity でも読める形式 (Markdown + frontmatter) であることを確認
- 動作実証は本 OSS 公開後に Antigravity ユーザによる検証を待つ

## 代替案

| 案 | 採否 | 理由 |
|---|---|---|
| A. **7 サイクル分割 (本案)** | 採用 | 各サイクル独立検証可能、Phase 5 と同流儀 |
| B. DX MCP アダプタを実 MCP 接続まで実装 | 却下 | Beta 仕様変動リスク、stub で十分 (本実装は Phase 7 で MCP GA 後) |
| C. npm publish を Phase 6 で実施 | 却下 | OSS 公開判断は Phase 7 で利用者検証後 |
| D. Antigravity 完全互換 | 却下 (Phase 7) | プラットフォーム個別検証コスト、本 OSS のスコープ外 |
