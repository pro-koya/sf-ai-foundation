## v0.3.0 (実施中)

**北極星**: 現参画プロジェクトで「資料更新が不要・ソースが正本」を週次運用で実証する。

**DoD (5 項目中 3 項目以上)**:
1. `yohaku sync` を 4 週連続で週次運用
2. 利用者主観「資料更新が不要」5 段階で 4 以上
3. 運用タスク AI 任せ可能比率 60% 以上
4. 1 機能あたりリリース準備時間 50% 短縮
5. 手動作業見逃し件数 0 / リリース

**制約**: 新機能追加なし、実利用と計測に集中。

## v0.4.0 (計画中)

**北極星**: 複数の Salesforce 案件に適用可能にし、改名移行 (`yohaku` → `yohakuforce` / `yohaku`) を完遂する。

**主要トピック**:
- **改名移行**: リポジトリ / パッケージ / CLI / ディレクトリ全更新
- AI プロバイダ抽象層 (Claude / OpenAI / Antigravity 選択可)

- v0.3.0 期間の `improvements/` 優先 3 件を吸収

## 設計原則 (変更しない)

- **3 層分離**: 決定的処理 / AI 判断 / 人手補完を混ぜない
- **1 バージョン = 1 North Star**: 隣のバージョンに目移りしない
- **正本は実装側**: Markdown は派生物
- **AI に生データを読ませない**: 知識グラフ経由
