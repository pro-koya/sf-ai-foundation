# /cycle-review

自律ループの「4. レビュー」ステップを実行する。

## 実行手順

1. **CRITICAL チェック** (即修正対象)
   - セキュリティ脆弱性 / 秘密情報漏洩
   - データ消失リスク
   - 禁則 10 か条違反
   → 1 件でもあれば即停止して報告

2. **HIGH チェック**
   - 設計 3 原則違反 (3 層分離 / 正本は実装側 / AI に生データを読ませない)
   - Phase ゴール逸脱・スコープクリープ
   - テスト不足 (80% 未満)

3. **MEDIUM チェック**
   - 関数 50 行超 / ファイル 800 行超 / ネスト 4 段超
   - マジックナンバー / 重複 / 不適切な命名
   - エラー握りつぶし

4. **専門 subagent 起動 (必要時)**
   - コードレビュー: `code-reviewer`
   - セキュリティ: `security-reviewer`
   - 並列で起動可能

5. **ナレッジ書き込み**
   - つまずいた点 → `.agents/knowledge/pitfalls/`
   - 効いた工夫 → `.agents/knowledge/wins/`
   - INDEX.md に 1 行追加

6. **報告**
   - CRITICAL / HIGH / MEDIUM の件数と対処方針を 5 行以内で報告

詳細は [`AGENTS.md` § 4. レビュー](../../AGENTS.md#4-レビュー-自己レビュー) を参照。
